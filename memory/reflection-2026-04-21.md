## 1. OPERATIONAL INEFFICIENCY

The Liverpool Q1 report revenue numbers were wrong twice before they were right once. The root cause was specific: I pulled numbers from Michael's meeting notes without cross-referencing the Klaviyo screenshot he had already provided. The screenshot was right there. I didn't use it.

The concrete fix isn't "verify data before reporting." It's this: when there are revenue numbers in both a meeting note and a Klaviyo screenshot, the screenshot wins. Always. And before any Callie pass on a report, I need to list the numbers I'm using and where they came from, then get a one-line confirm from Michael before Callie touches a single slide.

Same category of problem showed up with the Sunset Flow copy pass. Meaghan had left 28 comments in a Google Doc. I ran Callie on it without being able to see a single one. We burned a full Callie pass producing v2 that was based on nothing. The fix is already written in the memory file — download the .docx, unzip it, read `word/comments.xml` — but I need to make that a pre-flight step I never skip, not a lesson I remember to apply sometimes.
